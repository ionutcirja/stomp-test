Architectural decisions:

  • Model module: deal with data (add new item, sort items, compute mid price and mid price list for the last 30 seconds)
  • View module: render the table using the model data
  • Api module: connect to the server and subscribe to different topics
  • Main app controller: initiate the api, create the model, create the view and update the view when the model changes

Unit tests & linting:

  • Run npm test to see the tests running
  • I used mocha, chai, sinon and sinon-chai, nyc for coverage and babel-plugin-__coverage__ for better es6 coverage
  • Added some linting also (extend airbnb rules and modify some I don't really agree with)
